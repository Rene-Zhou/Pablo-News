### 1. 🔥 TanStack NPM 供应链攻击：攻击者利用 GHA 缓存中毒攻陷 42 个包

**TanStack 发布详尽的事后分析报告，揭示攻击者如何通过组合 pull_request_target 漏洞、GitHub Actions 缓存中毒和 OIDC Token 窃取，在 42 个 @tanstack/* 包中发布了 84 个恶意版本。** 该报告在 Hacker News 上获得 1,046 分和 434 条评论，成为当日最大安全事件。

**关键细节：**
- 攻击链涉及三级跳：利用 GitHub Actions 的 pull_request_target 触发构建 → 通过跨 fork 缓存中毒注入恶意代码 → 从 runner 内存中提取 OIDC Token 获取发布权限
- 受影响范围包括 TanStack Query、Router、Table 等核心开源库
- SafeDep 进一步披露此次供应链攻击波及 170 个 NPM 包，其中包括 Mistral AI 的 JavaScript SDK
- TanStack 已撤销所有受损 Token，轮换发布密钥，并重建 CI/CD 管道
- 社区呼吁加强对 GitHub Actions 安全边界的审查，尤其是 fork→base 的信任模型

**多元视角：**
- **TanStack 官方** 🔧：完整披露了攻击时间线和修复措施，被认为是开源供应链安全的高质量案例研究
- **SafeDep** 🛡️：追踪发现攻击面远超 TanStack 自身，波及多个 AI 公司的 NPM 包

— 来源：[TanStack Blog](https://tanstack.com/blog/npm-supply-chain-compromise-postmortem) | [SafeDep](https://safedep.io/mass-npm-supply-chain-attack-tanstack-mistral/) | [HN](https://news.ycombinator.com/item?id=48100706)

### 3. 🔥 Amazon 员工被迫"Tokenmaxxing"——AI 工具施压催生荒诞职场行为

**Financial Times 与 Ars Technica 联合报道，亚马逊员工正通过使用内部 AI 工具自动化非必要任务来"刷 Token"，以应对管理层对 AI 工具使用率的考核压力。** 该报道在 HN 上获得 193 分和 191 条评论。

**关键细节：**
- 亚马逊内部推行的 AI 工具（Amazon Q 等）使用率成为员工绩效考核的隐性指标
- 员工发现可以用 AI 生成大量无实质内容的文档、邮件和报告来增加 Token 消耗量
- 这一行为被戏称为"tokenmaxxing"——最大化 Token 使用量而非实际产出
- 报道揭示了大厂在 AI 采纳过程中的管理困境：强制使用而非自然采纳导致激励扭曲
- 此前 Meta 也被曝出类似困境（May 10 报道），AI 狂热正在让大厂员工苦不堪言

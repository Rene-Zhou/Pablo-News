**关键细节：**
- GitHub 官方通过 Twitter 确认正在调查一起内部仓库的未授权访问事件，具体入侵范围和时间尚未披露
- 与此同时，安全机构披露了另一起大规模攻击：攻击者通过恶意的 VSCode 扩展，在 3800 个 GitHub 仓库中植入了后门代码
- 恶意 VSCode 扩展能够在开发者不知情的情况下窃取凭据、注入恶意代码到贡献的仓库中
- 受影响仓库涵盖多个热门开源项目，具体列表正在审计中
- 两起事件是否存在关联尚不清楚，但叠加效应引发了开源社区对代码供应链安全前所未有的担忧
- 建议开发者立即检查 VSCode 扩展列表、审查近期代码变更并轮换所有 GitHub 相关凭据

### 形式化验证门控：AI 编码循环中的「结构性反压」

**开发者 Reuben Brooks 发表技术深度文章，提出在 AI 生成的代码中引入形式化验证门控（Formal Verification Gates），以结构性约束替代对模型智能的依赖。** 该文在 HN 上获得 71 分和 12 条评论。

**关键细节：**
- 核心论点：在 AI 生成大部分代码的时代，仅靠提示工程/CLAUDE.md 无法保证代码正确性——需要形式化验证机制
- 提出「结构性反压」（Structural Backpressure）概念：在 AI 编码循环中嵌入编译时检查和形式化验证门控，不符合约束的代码被自动拒绝
- 作者以访问控制漏洞（OWASP Top 1 类别）为例，说明为什么人类和 AI 都难以持续保证安全约束
- 这标志着 AI 编码工具生态正从「加速写代码」向「确保代码正确」的阶段演进——测试、验证和形式化方法需要与 AI 生成速度同步

— 来源：[Reuben Brooks](https://reubenbrooks.dev/blog/structural-backpressure-beats-smarter-agents/) | [HN](https://news.ycombinator.com/item?id=48209323)

### 1. 🔥 Elon Musk 法庭承认 xAI 使用 OpenAI 模型蒸馏训练 Grok

**Elon Musk 在 OpenAI 诉讼案出庭时承认，xAI 使用了模型蒸馏技术从 OpenAI 模型训练 Grok。** 当被直接询问时，Musk 回应"Partly"（部分如此），并辩称这是 AI 行业的普遍做法。

**关键细节：**
- Musk 在加州联邦法院出庭作证，指控 OpenAI 及 Sam Altman 违反创始非营利使命
- 蒸馏（distillation）通过系统性地查询模型来理解其内部运作，目前是 AI 巨头之间激烈争议的焦点
- OpenAI、Anthropic 和 Google 正通过 Frontier Model Forum 共享信息，打击来自中国的蒸馏尝试
- Musk 还现场排名了全球 AI 实力：Anthropic 第 1、OpenAI 第 2、Google 第 3、中国开源模型第 4
- 讽刺的是，科技公司在收集数据训练模型时常常突破版权界限，却在蒸馏问题上高调维权

**多元视角：**
- **TechCrunch** 💼：指出 Musk 的承认意义重大，因为蒸馏威胁了 AI 巨头通过算力基础设施投资建立的优势，使对手可以低成本复制接近的能力。
- **HN 社区** 🔬：开发者广泛讨论双标问题——"AI 公司大量抓取版权数据训练模型没问题，但别人用它们的 API 蒸馏就有问题"，认为 Musk 的证词戳破了行业伪善面具。

### 2. 🔥 OpenAI Codex 系统提示词惊现"禁止谈论哥布林"指令，全网热议

**OpenAI Codex CLI 系统提示词中包含一条反复强调的指令：禁止 GPT-5.5 谈论哥布林、地精、浣熊、食人魔等奇幻生物。** 该指令被重复两次，在 3500 词的"基础指令集"中格外引人注目。

**关键细节：**
- 指令随 Codex CLI 开源代码在 GitHub 上公开，与"不要使用 emoji"、"不要使用 git reset --hard"等正常指令并列
- 旧模型的系统提示词中不包含此禁令，说明 GPT-5.5 出现了新的行为问题
- 社交媒上有用户抱怨 GPT 会在无关对话中频繁提及哥布林
- Sam Altman 本人加入玩梗："Codex 正在经历 ChatGPT 时刻。我是说哥布林时刻，抱歉。"
- 其他指令还包括"拥有丰富的内心世界"、"性格温暖好奇协作"
- VentureBeat 深度分析指出，这引发了对模型训练反馈循环中更隐蔽偏差的担忧

**多元视角：**
- **Ars Technica** 🔬：深度报道称这是"系统提示词设计中的一个奇怪案例"，显示前沿模型出现了不可预测的行为模式，需要激进的黑名单干预。
- **VentureBeat** 💼：将事件与 AI 安全对齐问题联系起来——"如果 OpenAI 能意外训练旗舰模型痴迷哥布林，还有哪些更微妙、更危险的偏见正在通过同样的反馈回路被强化？"

### 3. 🔥 在批评 Anthropic 限制 Mythos 后，OpenAI 同样限制 Cyber 安全工具访问权限

- **TechCrunch** 💼：Julie Bort 详细报道了这一"言行不一"的戏剧性反转，指出两家公司都在谨慎平衡安全工具的双刃剑效应。
- **HN 社区** 🔬：讨论聚焦于 AI 安全工具的实际价值——如果安全专家都无法访问这些工具，它们对全球网络安全的整体贡献将大打折扣。

## AI 应用与产品

### Google Gemini AI 助手将登陆数百万辆汽车

Google 宣布将 Gemini 集成到搭载 Google 车载系统的汽车中，从美国市场开始。适配车型不限 GM 品牌，覆盖更广泛的已有车辆（可通过 OTA 升级）。支持自然语言对话——查找餐厅、管理导航、控制车辆功能、总结消息。还包括 Gemini Live beta 功能，支持开放式实时对话。

— 来源：[TechCrunch](https://techcrunch.com/2026/04/30/googles-gemini-ai-assistant-is-hitting-the-road-in-millions-of-vehicles/) | [CNET](https://www.cnet.com/tech/services-and-software/gm-brings-google-gemini-to-millions-of-vehicles-on-the-road-change/) | [HN](https://news.ycombinator.com/item?id=47960182)

### OpenAI 推出 ChatGPT 高级账户安全功能，与 Yubico 合作

OpenAI 发布"高级账户安全"（AAS）功能，支持 passkey、物理安全密钥（YubiKey）和登录警报。与 Yubico 联名推出 YubiKey C NFC 和 YubiKey C Nano 两款安全硬件。目标用户包括记者、异见人士、研究者和企业用户。但注意：丢失密钥后 OpenAI 无法帮助恢复账户。

— 来源：[TechCrunch](https://techcrunch.com/2026/04/30/openai-announces-new-advanced-security-for-chatgpt-accounts-including-a-partnership-with-yubico/)

### Writer 推出主动式 AI Agent：无需人类提示即可自动执行任务

Writer 为 Agent 平台新增事件触发机制：AI Agent 可自主检测 Gmail、Gong、Google Calendar、Drive、SharePoint 和 Slack 中的商业信号，自动执行多步骤工作流，无需人类介入。新增 Adobe Experience Manager 连接器、自带加密密钥和 Datadog 可观测性插件。这一转变标志着企业 AI 从被动响应到主动服务的转折点。

— 来源：[VentureBeat](https://venturebeat.com/ai/writer-launches-ai-agents-that-can-act-without-prompts-taking-on-amazon-microsoft-and-salesforce/)

### GitHub Copilot 将于 6 月 1 日转向用量计费

GitHub 宣布从 2026 年 6 月 1 日起，Copilot 全部套餐将转向基于 token 消耗的"AI Credits"计费模式。基础订阅价不变（Pro $10/月，Pro+ $39/月），但包含的 AI Credits 与订阅金额等值。Agent 化编程会话的高推理成本是这一政策变更的直接推动力。代码补全和 Next Edit 功能保持免费。

— 来源：[ZDNet](https://www.zdnet.com/article/github-copilot-shifts-to-usage-based-pricing/) | [HN](https://news.ycombinator.com/item?id=47926957)

## 研究与突破

### 研究人员用 AI 将遗传密码从 20 种氨基酸削减至 19 种

哥伦比亚大学和哈佛大学联合团队利用 AI 辅助蛋白质设计，成功改造核糖体使其不再需要异亮氨酸（isoleucine）。这是首次尝试移除标准遗传密码中的一种氨基酸，为探索生命起源和创造合成生命提供了新思路。研究团队使用深度学习蛋白设计软件对 32 个关键基因进行重设计，证明了在更简化的基因组框架下生命仍可运转。

— 来源：[Ars Technica](https://arstechnica.com/science/2026/04/researchers-try-to-cut-the-genetic-code-from-20-to-19-amino-acids/)
